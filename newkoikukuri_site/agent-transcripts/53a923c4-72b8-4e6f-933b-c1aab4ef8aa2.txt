+# Koi-Kukuri × Meta-Life オンライン婚活サイト & 管理画面
+
+「がんばらなくても、つながれる出会い」をコンセプトにした、オンライン仮想空間「Meta-Life」を利用した婚活イベント用の Web サイトと管理画面です。
+
+## 構成
+
+- 公開サイト
+  - トップヒーローエリア（イベントコンセプト）
+  - 3つの特徴
+  - 参加にあたっての注意点
+  - イベント情報一覧（管理画面で登録）
+  - お知らせ一覧（管理画面で登録）
+  - 運営情報 / 運営からの一言
+- 管理画面 `/admin`
+  - イベント登録（題名 / 概要 / 開催日時 / URL / サムネイル画像 ※任意）
+  - トップページ画像アップロード
+  - お知らせ登録（題名 / 概要 / 掲載日時）
+
+## 動作環境
+
+- Node.js 18 以降を推奨
+
+## セットアップ
+
+```bash
+npm install
+npm start
+```
+
+起動後、以下にアクセスします。
+
+- 公開サイト: `http://localhost:3000/`
+- 管理画面: `http://localhost:3000/admin`
+
+## 管理画面の使い方
+
+- **イベント登録**
+  - 題名・概要・開催日時・申込 URL を入力し、必要であればサムネイル画像をアップロードして「イベントを登録する」を押します。
+  - 登録したイベントは公開サイトの「イベント情報」に新しい順で表示されます。
+- **トップ画像設定**
+  - トップページに表示したい横長画像を選択し「トップ画像を更新する」を押します。
+  - 未設定の場合は、デフォルトのグラデーション背景が表示されます。
+- **お知らせ登録**
+  - 題名・概要・掲載日時を入力して「お知らせを登録する」を押します。

## 2. 代表的なリリース方法

### A. VPS やレンタルサーバーに自分でデプロイ

**例: ConoHa VPS、さくらのVPS、AWS EC2 など**

1. サーバーを借りる
2. Node.js をインストール
3. プロジェクトを `git clone` またはファイル転送
4. `npm install` → `npm start`
5. **PM2** や **systemd** で常時起動に設定
6. Nginx や Apache でリバースプロキシを設定し、HTTPS 化

**メリット**: サーバーを自由に使える  
**デメリット**: 設定や運用に多少知識が必要

---

### B. PaaS（Platform as a Service）を使う

**例: Railway、Render、Heroku、Cyclic など**

1. GitHub にプロジェクトを push
2. サービスにアカウント登録して GitHub と連携
3. リポジトリを選択してデプロイ
4. 環境変数で `PORT` などを設定（サービスによる）

**メリット**: 設定が少なく、デプロイが楽  
**デメリット**: 無料枠の制限や、有料だと月額コストがかかる

---

### C. 静的サイトホスティング + サーバーレス API

- フロント（HTML/CSS）を **Vercel** や **Netlify** にデプロイ
- イベント・お知らせ・画像管理などのAPIを **Cloudflare Workers** や **AWS Lambda** で動かす

**メリット**: スケールしやすく、運用コストを抑えやすい  
**デメリット**: 現在の構成を大きく作り変える必要がある

## 3. このプロジェクトの構成に合うやり方

| 方法 | 難易度 | 目安コスト |
|------|--------|------------|
| **Railway / Render** にデプロイ | ⭐ 簡単 | 無料枠あり、以降は月数百円〜 |
| **VPS + PM2 + Nginx** | ⭐⭐ やや難しい | 月500円〜 |

## 6. 新しいリポジトリに push する

```powershell
cd C:\Users\s0666\workspace\newkoikukuri_site
git status
git add .
git commit -m "Webサイト・管理画面を移行"
git push origin main
```

- ブランチ名が `master` の場合は、最後の行を `git push origin master` にします。  
- `git status` で「nothing to commit」と出る場合は、すでにコミット済みなので `git push origin main`（または `master`）だけ実行すればOKです。

## 7. 移行後の公開（Render などで URL 共有する場合）

- 公開先を **Render** にする場合は、  
  「Connect a repository」で **kokioshima73118/newkoikukuri_site** を選び、  
  **Root Directory** は空欄（ルートにサイトがあるため）、**Start Command** は `npm start` のままにします。  
- 以前「koikukuri/main」の `server` フォルダでデプロイしていた場合は、Render のサービス設定を **newkoikukuri_site** のルートに切り替えるか、新しく Web サービスを作り直す形になります。
